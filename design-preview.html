待确认的决策点

❓ 决策 1
"Skills & Gears" 拆开还是合并？
Skills 已有 15 个详情页。Abyss Gears 数据还没做。合并在一起可以省一个菜单位（8 变 7），但概念上它们是两个系统。拆开的话就是 9 个菜单 — 你觉得能接受 9 个吗？
❓ 决策 2
Crafting / Cooking / Mounts 放哪？
这三个现在在首页 Hub Grid 里，没有对应的顶部菜单。选项：① 在首页保留卡片，导航不设菜单；② 归入 Guides 门下（Crafting Guide）；③ 合并为一个 "More" 或 "Wiki" 菜单。你倾向哪个？
❓ 决策 3
News / Reviews 放在哪？
首页 Hero 下面可以有一条「Latest Updates」资讯横条（3-4 篇最新的 patch notes / reviews）。但没有单独的顶部菜单入口。需要给它一个 "News" 菜单吗，还是走首页卡片 + Guides 里的 Blog 区？
❓ 决策 4
详情页模板优先级
Type A/B（数据库列表+详情）已有，直接复用。Type C/D/E/F 四种新模板，开发顺序你希望怎么排？建议：Type D（Guide）→ Type C（Map）→ Type E（Tool）→ Type F（Region）。
✅ 我的建议
先确认导航，再定模板优先级
1. 这 8 个菜单名称你看着顺不顺？有没有要改的？
2. 如果菜单定了，首页布局基本不用大改 — 现有 Hub Grid + Regions + Systems 结构直接映射到新菜单。
3. 模板 Type A/B 不用动（已有），Type D（攻略文章）先做一个出来跑通流程，再推 Type C（地图）和 Type E（工具）。
